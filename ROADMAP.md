# Trivia จับเวลา — Feature & Growth Roadmap

เอกสารนี้วางแผนการพัฒนาเกม โดยจัดลำดับตาม **ผลต่อการเติบโตของผู้เล่น (impact) เทียบกับแรงที่ต้องลง (effort)**

## สถานะปัจจุบัน (Baseline)

สิ่งที่มีแล้ว:
- เล่นได้ 1 หมวด (ความรู้ทั่วไป), อีก 3 หมวดล็อกไว้ "เร็วๆนี้"
- 10 คำถาม, จับเวลา 10 วินาที/ข้อ
- คะแนนตามความเร็ว (100–200 แต้ม/ข้อ)
- เสียงประกอบ, หน้าสรุปผล, ดูเฉลย, แชร์คะแนน
- ภาษาไทย, mobile-first, ดีไซน์ pastel

ช่องว่างหลักที่ฉุดการเติบโต:
- **ไม่มีเหตุผลให้กลับมาเล่นซ้ำ** — เล่นจบ 10 ข้อแล้วจบเลย, คำถามชุดเดิมทุกครั้ง
- **ไม่มี backend** — คะแนนไม่ถูกบันทึก, ไม่มี leaderboard, ไม่มีบัญชี
- **loop การแชร์อ่อน** — แชร์ได้แค่ข้อความ ไม่มีการท้าเพื่อน
- **คอนเทนต์น้อย** — คำถามหมดเร็ว

---

## กรอบคิด: Growth Loop

โฟกัสที่ 5 ด่านของ funnel:
1. **Acquisition** — คนใหม่เจอเกมได้ยังไง
2. **Activation** — ประสบการณ์เล่นครั้งแรกดีไหม
3. **Retention** — ทำไมต้องกลับมาเล่นอีก ← *คันโยกสำคัญที่สุดตอนนี้*
4. **Referral** — ทำไมต้องชวนคนอื่น
5. **Revenue** — (ทีหลัง เมื่อมี traction แล้ว)

---

## Phase 0 — รากฐาน (Foundation) · 1–2 สัปดาห์

ต้องมีก่อน ไม่งั้น feature อื่นต่อยอดไม่ได้ และวัดผลไม่ได้

| Feature | ทำไม | Effort |
|---|---|---|
| **สุ่มคำถาม + สลับตัวเลือก** ทุกเกม | เล่นซ้ำได้ไม่ซ้ำเดิม — quick win ที่ impact สูงมาก | ต่ำ |
| **ขยายคลังคำถาม** (จาก 10 → 100+ ข้อ ต่อหมวด) | เล่นได้นานขึ้น, สุ่มได้จริง | ต่ำ–กลาง |
| **บันทึกข้อมูลใน localStorage** (best score, จำนวนเกม, streak) | ให้ผู้เล่นเห็นพัฒนาการ โดยยังไม่ต้องมี backend | ต่ำ |
| **PWA / Add to Home Screen** | ติดตั้งบนมือถือได้เหมือนแอป, เล่น offline ได้, ดัน retention บนเว็บมาก | กลาง |
| **Analytics** (เช่น Plausible / GA4 / PostHog) | ต้องรู้ว่าคนหลุดตรงไหน ก่อนตัดสินใจทำอะไรต่อ | ต่ำ |

**เป้าหมายเฟสนี้:** วัดผลได้ + เล่นซ้ำแล้วสนุก

---

## Phase 1 — Retention: ทำให้กลับมาเล่น · 2–4 สัปดาห์

คันโยกสำคัญที่สุดสำหรับการโตอย่างยั่งยืน

| Feature | ทำไม | Effort |
|---|---|---|
| **Daily Challenge (โจทย์ประจำวัน)** | ทุกคนเจอชุดคำถามเดียวกันในแต่ละวัน → สร้างนิสัยกลับมาเล่นทุกวัน + แชร์ "วันนี้ได้เท่าไหร่" | กลาง |
| **ระบบ Streak (เล่นต่อเนื่องกี่วัน)** | แรงจูงใจให้ไม่ขาดวัน (แบบ Duolingo/Wordle) | ต่ำ |
| **ปลดล็อกหมวดที่ล็อกไว้ให้เล่นจริง** (กีฬา/บันเทิง/วิทยาศาสตร์) | มีคอนเทนต์รออยู่แล้วในดีไซน์ แค่เติมคำถาม | ต่ำ–กลาง |
| **ระดับความยาก / Adaptive difficulty** | ง่าย→ยากขึ้นเรื่อยๆ ให้ท้าทายพอดี ไม่เบื่อ ไม่ท้อ | กลาง |
| **Achievements / เหรียญตรา** | เป้าหมายย่อยให้ไล่เก็บ (เช่น ตอบถูก 10 ข้อรวด, เล่นครบ 7 วัน) | กลาง |

**เป้าหมายเฟสนี้:** D1/D7 retention ขึ้น, มีเหตุผลกลับมาทุกวัน

---

## Phase 2 — Social & Viral: ให้ผู้เล่นพาคนมาเอง · 4–8 สัปดาห์

ต้องมี backend เบาๆ (เช่น Supabase / Firebase) เพื่อเก็บคะแนน

| Feature | ทำไม | Effort |
|---|---|---|
| **ท้าเพื่อน (Challenge a Friend)** | ส่งลิงก์ที่มีชุดคำถามเดียวกัน แล้วเทียบคะแนน — **viral loop ที่แรงที่สุด** (K-factor) | กลาง |
| **Leaderboard** (รวม + เฉพาะเพื่อน + รายสัปดาห์) | การแข่งขันดึงให้กลับมาไต่อันดับ | กลาง–สูง |
| **การ์ดแชร์เป็นรูปภาพ + OG tags** | แชร์แล้วสวย เห็นคะแนนเป็นภาพ → คนคลิกเข้ามาเล่นมากขึ้น | กลาง |
| **ระบบบัญชี / login เบาๆ** | เก็บ progress ข้ามอุปกรณ์, จำเป็นสำหรับ leaderboard จริง | กลาง |

### 🇹🇭 เจาะตลาดไทยโดยเฉพาะ — LINE
- **แชร์เข้า LINE / LINE OpenChat** — ช่องทางไวรัลอันดับ 1 ในไทย
- **LINE Login** — สมัคร/ล็อกอินง่าย คนไทยคุ้นเคย
- **LINE Mini App** — รันเกมในตัว LINE เลย ลดแรงเสียดทานการเข้าถึงมหาศาล

**เป้าหมายเฟสนี้:** ผู้เล่นชวนกันเอง (K-factor > 0.5), มีการแข่งขัน

## Phase 3 — Content & Platform Scale · ต่อเนื่อง

| Feature | ทำไม | Effort |
|---|---|---|
| **คำถามจากชุมชน (UGC)** + ระบบตรวจสอบ | คลังคำถามโตเองโดยไม่ต้องเขียนเองทั้งหมด | สูง |
| **ชุดคำถามตามเทศกาล/กระแส** (สงกรานต์, บอลโลก, ข่าวดัง) | คอนเทนต์สดใหม่ ดึงคนช่วงพีค | กลาง |
| **โหมดแข่งเรียลไทม์ (1v1 / ห้องหลายคน)** | เล่นกับเพื่อนสดๆ engagement สูงมาก | สูง |
| **Localization (อังกฤษ + ภาษาอื่น)** | ขยายตลาดออกนอกไทย | กลาง |

---

## Phase 4 — Monetization (เมื่อมี traction แล้วเท่านั้น)

อย่าใส่เร็วเกินไป จะฉุดการเติบโต ทำเมื่อ retention นิ่งแล้ว

- **Rewarded ads** — ดูโฆษณาแลกไอเทม (เล่นซ้ำ Daily, ใบ้คำตอบ)
- **Remove ads / Premium** — จ่ายครั้งเดียวหรือรายเดือน
- **Power-ups** — เพิ่มเวลา, ข้ามข้อ, ใบ้ (50:50)
- **ชุดคำถามพรีเมียม** — หมวดพิเศษ

## Metrics ที่ต้องจับตา

| Metric | ความหมาย |
|---|---|
| **DAU / MAU** | ผู้เล่นรายวัน/รายเดือน |
| **D1 / D7 / D30 Retention** | กลับมาเล่นหลัง 1/7/30 วัน — ตัวชี้วัดสุขภาพเกมที่สำคัญที่สุด |
| **Session length & games/session** | เล่นนานแค่ไหน กี่เกมต่อครั้ง |
| **K-factor** | ผู้เล่น 1 คนชวนคนใหม่ได้กี่คน (> 1 = โตแบบไวรัล) |
| **Daily Challenge completion rate** | สัดส่วนคนเล่นโจทย์ประจำวัน |
| **Share rate** | สัดส่วนคนที่กดแชร์หลังจบเกม |

## สรุปลำดับความสำคัญ (ถ้าเลือกทำได้ทีละอย่าง)

1. 🥇 **สุ่มคำถาม + ขยายคลัง** — แก้ปัญหา "เล่นซ้ำไม่ได้" ที่ฉุดทุกอย่าง (effort ต่ำ, impact สูง)
2. 🥈 **Daily Challenge + Streak** — สร้างนิสัยกลับมาเล่นทุกวัน
3. 🥉 **ท้าเพื่อน + แชร์เข้า LINE** — ให้ผู้เล่นพาคนใหม่มาเอง
4. **Leaderboard + บัญชี** — เติมแรงจูงใจการแข่งขันระยะยาว

> **ข้อสมมติที่ใช้:** เกมยังเป็น web/PWA เป็นหลัก, ตลาดหลักคือไทย, ยังไม่รีบทำรายได้
> ถ้าเป้าหมายต่างจากนี้ (เช่น จะทำเป็น native app, เจาะตลาดต่างประเทศ, หรือเน้นรายได้เร็ว) บอกได้ เดี๋ยวปรับลำดับให้
